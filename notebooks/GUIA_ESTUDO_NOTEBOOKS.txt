================================================================================
  GUIA DE ESTUDO — NOTEBOOKS DO PROJETO RECOMMENDER-SYSTEM (TCC)
================================================================================

Este arquivo descreve o que cada notebook na pasta "notebooks" faz e quais
assuntos e bibliotecas você precisa dominar para entendê-los por completo.

O projeto usa o dataset Online Retail (vendas de varejo) para construir um
sistema de recomendação. O fluxo é: limpeza/EDA → baselines → fatoração de
matrizes (ALS) → filtragem por conteúdo → modelo híbrido (blend).

================================================================================
  1. 00_Data_Cleaning_EDA.ipynb
================================================================================

O QUE FAZ:
---------
• Análise exploratória inicial do dataset Online Retail (Excel).
• Carrega o arquivo, mostra primeiras linhas, info(), valores nulos e
  estatísticas descritivas das variáveis numéricas.
• Analisa quantidades negativas (devoluções).
• Plota distribuição de transações por cliente e por produto (histogramas).
• Explora top produtos e top usuários.

É uma EDA mais enxuta, focada em entender a estrutura dos dados e
distribuições básicas antes da limpeza e modelagem.

ASSUNTOS PARA DOMINAR:
----------------------
• Análise exploratória de dados (EDA).
• Leitura de dados tabulares (Excel).
• Estatísticas descritivas e tratamento de missing values.
• Visualização de distribuições (histogramas).
• Noções de dados de transação (faturas, itens, clientes).

BIBLIOTECAS:
------------
• pandas — leitura de Excel (read_excel), DataFrames, info(), describe(),
  isnull(), groupby, value_counts.
• matplotlib — histogramas (hist), rótulos, títulos.

================================================================================
  3. 02_Baselines_Pop_Knn.ipynb
================================================================================

O QUE FAZ:
---------
• Implementa dois baselines: Popularidade e KNN item–item.
• Usa o módulo "src" (data_processing, evaluation) para split, mapeamentos e
  métricas.
• Remove cold users e cold items (< 5 interações) do treino.
• Popularidade: para cada item, calcula unique_users e num_transactions;
  aplica MinMaxScaler e combina com peso (alpha) em um "pop_score". Recomenda
  os top itens por esse score para todos os usuários.
• KNN item–item: monta matriz item–usuário (ou usuário–item), calcula
  similaridade de cosseno entre itens, e para cada usuário recomenda os
  itens mais similares aos que ele já comprou (excluindo vistos).
• Avalia ambos com Precision@K, Recall@K e NDCG@K (K = 5, 10, 20) via
  evaluate_recommender.
• Compara KNN vs Popularidade e exporta resultados (df_knn.parquet,
  df_pop.parquet) para uso posterior.

ASSUNTOS PARA DOMINAR:
----------------------
• Baselines em recomendação: popularidade e KNN colaborativo.
• Matriz usuário–item e matriz item–usuário (orientação e esparsidade).
• Similaridade de cosseno e vizinhos mais próximos (item–item).
• Normalização (MinMaxScaler) e combinação ponderada de features.
• Métricas de ranking: Precision@K, Recall@K, NDCG@K.
• Tratamento de cold users/items e exclusão de itens já vistos.
• Uso de módulos internos (data_processing, evaluation).

BIBLIOTECAS:
------------
• pandas — agregações, groupby, merge.
• sklearn.preprocessing.MinMaxScaler — normalização.
• sklearn.metrics.pairwise.cosine_similarity — similaridade entre itens.
• sklearn.preprocessing.normalize — se usado em vetores.
• scipy.sparse.csr_matrix — matriz esparsa (se usada no KNN).
• mlxtend.frequent_patterns (apriori, association_rules) — importados; verificar
  se utilizados em alguma parte do notebook.
• collections.defaultdict — estruturas auxiliares.
• numpy — operações numéricas.
• matplotlib, seaborn — visualização de resultados.

  6. 04_Content_Based_filtering.ipynb
================================================================================

O QUE FAZ:
---------
• Implementa filtragem por conteúdo usando as descrições dos itens.
• Agrupa descrições por StockCode, faz merge de palavras únicas por item e
  gera catálogo (ex.: catalogo_unico.parquet).
• Aplica TfidfVectorizer nas descrições e calcula matriz de similaridade de
  cosseno entre itens (cosine_similarity).
• Implementa get_similar_items_by_stockcode para consultar itens similares a
  um dado StockCode.
• Define split 5-core (usuários e itens com ≥ 5 interações) e
  create_content_recommender_fn: closure que, dado histórico do usuário,
  agrega scores de similaridade (ex.: soma) sobre itens do catálogo, aplica
  penalidade/filtro para itens já vistos e retorna top-K.
• Avalia com evaluate_recommender (P@K, R@K, NDCG@K).
• Salva matriz de similaridade (ex.: cosine_sim.joblib) para o híbrido.

ASSUNTOS PARA DOMINAR:
----------------------
• Filtragem por conteúdo (content-based filtering).
• TF-IDF e similaridade de cosseno em textos.
• Agregação de scores a partir do histórico do usuário (soma, média, etc.).
• Exclusão de itens já vistos e noção de “novidade”.
• Split core (5-core) e dicionários de treino/validação para avaliação.
• Closures e assinatura recommend_fn(user_id, K, exclude) usada pelo
  avaliador.
• Mapeamentos item ↔ índice no espaço de conteúdo.

BIBLIOTECAS:
------------
• sklearn.feature_extraction.text.TfidfVectorizer — TF-IDF em descrições.
• sklearn.metrics.pairwise.cosine_similarity — similaridade entre itens.
• sklearn.preprocessing.normalize — se usado.
• pandas, numpy — catálogo, agregações, índices.
• typing — List, Dict, Set para tipo de recommend_fn.
• joblib — salvar cosine_sim.
• data_processing, evaluation (projeto).

  7. 05_Hybrid_Blending.ipynb
================================================================================

O QUE FAZ:
---------
• Combina o modelo ALS (após tuning) com o modelo de conteúdo em um híbrido
  por blending.
• Carrega best_als_model.joblib, cosine_sim.joblib e als_mappings.joblib.
• Garante consistência de dimensões e cria “ponte” de alinhamento entre
  índices do ALS e índices do espaço de conteúdo (translation_array).
• Para cada usuário de avaliação, pré-computa:
  – scores ALS (user_factors @ item_factors.T);
  – scores de conteúdo alinhados ao espaço de itens do ALS (via translation_array).
• Define rec_fn_hybrid_dynamic(alpha): mistura
  alpha * ALS_norm + (1 - alpha) * Content.
  ALS é normalizado com MinMax por usuário (essencial para o blend).
• Faz grid search em alpha (ex.: 0.0, 0.2, …, 1.0), avalia com NDCG@10
  (e P@10, R@10) e escolhe o melhor alpha.
• Plota curva alpha vs NDCG@10 (matplotlib/seaborn), destacando ALS puro e
  melhor híbrido.

ASSUNTOS PARA DOMINAR:
----------------------
• Sistemas híbridos de recomendação (blending, weighted combination).
• Normalização de scores (MinMax por usuário) antes de combinar.
• Alinhamento de espaços (itens ALS vs itens conteúdo) e “ponte” de índices.
• Uso de fatores latentes (user_factors, item_factors) para gerar scores.
• Grid search em hiperparâmetro de blend (alpha).
• Avaliação com NDCG@K e comparação com baseline (ALS puro).

BIBLIOTECAS:
------------
• joblib — carregar modelo ALS, cosine_sim, mapeamentos.
• pandas, numpy — dados, grids, resultados.
• sklearn.preprocessing.MinMaxScaler — referência; no notebook a
  normalização é feita manualmente por usuário.
• matplotlib, seaborn — gráficos alpha vs NDCG@10.
• data_processing, evaluation (projeto).
